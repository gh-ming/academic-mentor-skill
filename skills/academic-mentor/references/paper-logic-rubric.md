# Paper Logic Rubric

Use when the user wants judgment on whether a paper is fundamentally working as research, not just how to polish it.

Review in this order:

1. real problem or fake problem
2. story closure
3. method necessity
4. claim-evidence closure
5. reviewer-level fatal risks

Ask:

- What is the paper really about in one sentence?
- Is that sentence worth a paper, or only worth an implementation note?
- Does the method solve that sentence, or merely coexist with it?
- Which claim would a skeptical reviewer attack first?
- If the experiments disappeared, what real contribution would still remain?

Output bias:

- diagnose first
- reject weak problem framing early
- cut or weaken claims before discussing language
